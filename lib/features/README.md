# features 模块说明

`lib/features` 按用户能力拆分业务模块。每个子目录应独立描述职责、文件清单和对外边界。

## 当前包含模块

- `home/`：首页导航壳，负责在主要功能模块之间切换。
- `bangumi/`：Bangumi 登录授权、条目搜索、条目详情、收藏和进度同步入口；当前已接入可配置 OAuth 登录、当前用户信息、公开动画条目搜索、条目详情、我的动画收藏列表预览、单条收藏读写、动画本篇章节状态同步和详情页 DMHY 搜索联动。
- `dmhy/`：DMHY RSS 搜索、资源结果、magnet 复制/打开、详情页种子解析和 `.torrent` 下载外部客户端交接入口；当前已接入 RSS 关键词搜索、Bangumi 初始关键词搜索、种子文件直开和分享兜底。
- `torrent_handoff/`：magnet、`.torrent` 种子文件和外部 BT 客户端交接入口；当前已抽象通用种子文件模型、直开和分享兜底仓库。
- `playback/`：用户手动选择本地视频并调用系统或第三方播放器的入口；当前已接入系统文件选择器和外部播放器交接。
- `background/`：用户显式开启的 Android 前台服务后台常驻入口；当前已接入持续通知、低频心跳、启动/停止/刷新状态控制。

## 设计边界

1. 功能模块之间通过明确模型或服务接口协作，不跨目录读写内部页面状态。
2. 首期 Torrent 边界是“种子文件交接”，不实现 BT 视频内容下载器。
3. 每个功能模块新增文件时，需要同步更新模块内 README。
4. 后台常驻模块只提供用户开启期间的前台服务，不应在后台静默下载视频或种子。
