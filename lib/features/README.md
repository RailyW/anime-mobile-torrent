# features 模块说明

`lib/features` 按用户能力拆分业务模块。每个子目录应独立描述职责、文件清单和对外边界。

## 当前包含模块

- `home/`：首页导航壳，负责在主要功能模块之间切换。
- `bangumi/`：Bangumi 登录授权、条目搜索、条目详情、收藏和进度同步入口；当前已接入可配置 OAuth 登录、本机 OAuth client 设置页、当前用户信息、公开动画条目搜索、输入防抖、搜索排序、搜索结果分页加载更多、读取请求 429 退避、条目详情、搜索结果 DMHY 搜索联动、我的动画收藏分页列表、收藏列表 DMHY 搜索联动、单条收藏读写、动画章节状态同步、章节类型筛选、已加载章节展开查看、章节分页加载更多、批量标记到第 N 话看过和详情页 DMHY 搜索联动。
- `dmhy/`：DMHY RSS 搜索、资源结果、magnet 复制/打开、详情页种子解析和 `.torrent` 下载外部客户端交接入口；当前已接入 RSS 关键词搜索、Bangumi 搜索结果/收藏列表/条目详情初始关键词搜索、搜索结果一键保存为订阅关键词、RSS 与种子请求 429 退避、资源标题元数据标签、HTML 列表真实大小与资源热度统计、前台资源排序、包含片源、字幕说明、字幕语言、最小种子数和排除关键词的前台资源筛选、字幕组偏好保存与自动套用、资源卡片外部 BT 客户端预提示、按检测结果动态调整主按钮、APP 专属持久种子目录、种子文件最近记录、直开和分享兜底。
- `subscriptions/`：DMHY RSS 订阅检查入口；当前已接入本地关键词保存、DMHY 搜索结果页一键保存当前关键词、动画分类/全站范围、手动 RSS 检查、后台低频自动检查、新命中识别、最近结果摘要展示、失败原因展示和回流 DMHY 搜索。
- `torrent_handoff/`：magnet、`.torrent` 种子文件和外部 BT 客户端交接入口；当前已抽象通用种子文件模型、直开和分享兜底仓库、Android 系统文档创建器导出仓库，接入 Android resolver 外部客户端能力检测和候选客户端展示、最近种子记录、本地种子文件清理、本机兼容实测记录与兼容报告复制，并在页面中提供外部 BT 客户端兼容自检、失败处理引导和前往播放页手动选择视频的闭环入口。
- `playback/`：用户手动选择本地视频并调用系统或第三方播放器的入口；当前已接入系统文件选择器、最近视频本机记录、单条最近记录删除和外部播放器交接。
- `background/`：用户显式开启的 Android 前台服务后台常驻入口；当前已接入持续通知、低频心跳、通知点击回到后台页或新订阅命中时直达 DMHY 搜索、重复命中回后台摘要页、通知查看后台按钮、通知停止按钮、启动/停止/刷新状态控制。

## 设计边界

1. 功能模块之间通过明确模型或服务接口协作，不跨目录读写内部页面状态。
2. 首期 Torrent 边界是“种子文件交接”，不实现 BT 视频内容下载器。
3. 每个功能模块新增文件时，需要同步更新模块内 README。
4. 后台常驻模块只提供用户开启期间的前台服务，不应在后台静默下载视频或种子。
5. 订阅检查模块只保存用户显式添加的关键词并按需检查 RSS；后台低频自动检查必须依赖用户显式启动前台服务，且不得自动下载种子或视频。订阅命中的后续操作应通过显式跳转到 DMHY 搜索页完成。
