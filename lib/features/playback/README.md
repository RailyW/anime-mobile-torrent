# playback 模块说明

`lib/features/playback` 负责播放入口：用户手动选择本地视频文件、保存本机最近选择记录，并调用系统或第三方播放器。

## 当前包含文件

- `domain/local_video_file.dart`：用户授权选择的本地视频文件模型，包含路径、文件名、MIME 类型、文件大小和常见视频 MIME 推断工具。
- `domain/recent_local_video.dart`：最近选择视频记录模型，保存用户显式选择过的视频元信息和选择时间，用于在播放页快速回选。
- `domain/playback_open_result.dart`：外部播放器打开结果模型，将平台返回值转换成播放模块内部状态和中文提示。
- `application/playback_providers.dart`：播放仓库接口、最近视频记录仓库接口、基于 `file_selector`、`open_filex` 的播放实现，以及基于 `SharedPreferences` 的最近视频本机记录实现，负责编排选择视频、记录最近选择与调起播放器。
- `presentation/playback_tab.dart`：播放首页入口，提供选择视频、查看已选文件信息、最近视频回选、清空最近记录、调起系统或第三方播放器和展示打开结果。

## 设计边界

1. 首期无法自动知道外部 BT 客户端下载出来的视频路径，需要用户主动选择本地视频；最近视频只保存用户已显式选择过的记录。
2. 播放模块只处理用户授权的文件信息，不扫描外部下载目录，不申请全文件访问权限。
3. 如果后续实现内置下载器，播放模块可以接收下载器产出的可播放文件列表。
4. 当前不实现内嵌视频播放器，不解析视频编码，也不接管播放进度；实际播放能力由用户手机上安装的播放器决定。
5. 最近视频记录只保存在本机，旧路径可能因为系统权限、外部文件移动或删除而失效，真实可播放性以打开动作返回结果为准。

## 使用的成熟套件

- `file_selector`：调用系统文件选择器，让用户显式选择本地视频文件。
- `open_filex`：通过平台能力打开文件，把视频交给系统或第三方播放器。
- `shared_preferences`：保存少量本机最近视频记录，不引入数据库或外部同步。
