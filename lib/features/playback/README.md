# playback 模块说明

`lib/features/playback` 负责播放入口：用户手动选择本地视频文件，并调用系统或第三方播放器。

## 当前包含文件

- `presentation/playback_tab.dart`：播放首页入口，目前展示本地视频选择和播放器调起能力的接入状态。

## 后续文件规划

- `domain/`：本地视频文件、MIME 类型和播放器打开结果模型。
- `application/`：文件选择、URI 校验、错误提示和播放命令编排。
- `platform/`：必要时封装 Android 播放 Intent 和 URI 授权逻辑。
- `presentation/`：本地视频选择按钮、最近打开记录和播放失败反馈。

## 设计边界

1. 首期无法自动知道外部 BT 客户端下载出来的视频路径，需要用户主动选择本地视频。
2. 播放模块只处理用户授权的 URI，不扫描外部下载目录，不申请全文件访问权限。
3. 如果后续实现内置下载器，播放模块可以接收下载器产出的可播放文件列表。
