# AGENTS.md instructions

## 语言要求

保持以中文回答用户的任何问题。

## 命令要求

除非用户明确要求，否则以下指令应当严禁使用：

* git commit
* git switch
* git checkout
* git push

用户对于自身的仓库的 git 树有绝对的所有权。在获得用户的授权前，严禁以任何形式修改 git 树的内容。

用户对自己所处的分支有绝对的选择权，在 main 分支上做修改是被允许的。

用户已授权：每个功能结束之后，都允许自动执行提交并推送到远程；所有后续小功能开发完成后，也允许自动提交并推送到远程。

注意，当前运行在 Windows 环境下。Windows 对于大文件补丁写入兼容不佳，需要避免出现大文件一次性补丁写入问题。对于长文本应当分段写入。

## 代码编写要求

写入代码时，每个独立的模块，应当有完整的、详细描述功能以及包含文件的相关表述的 README。当修改了模块内文件时，应当同时修改 README，以同步更新。

代码应当从函数声明、接口契约，到函数内逻辑，都应当有详尽清晰的中文注释，以方便开发者了解结构内容。

## 子代理相关要求

当需要使用子代理进行工作时，优先使用 gpt-5.5 xhigh 作为子代理参数配置。

## TDD 相关要求

对一切前端工程，除非明确要求，否则不遵循 superpowers 的对于 TDD 的相关要求，不做 TDD，直接进行代码变更和功能更新。

非必要不进行全量测试。对于全量测试，一旦出现全量测试超时，严禁扩大时间窗口后重跑。必须转为分模块重跑。对于功能完全不交叉、不会引起共享临时文件冲突的，可以使用 subagent 并行运行测试。
